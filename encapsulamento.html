Encapsulamento em Python

Controlando o acesso aos dados com métodos get/set

Princípio fundamental da POO: Proteger os dados internos e controlar como eles podem ser acessados e modificados.

O que é Encapsulamento?

Definição:
Princípio fundamental da POO que:

  • Protege os dados internos de uma classe
  • Controla como esses dados podem ser acessados e modificados
  • Organiza o código em uma interface clara e segura

Analogia: Como um cofre bancário

  • Você não acessa o dinheiro diretamente
  • Usa métodos controlados (como uma combinação) para depositar/retirar

Modificadores de Acesso em Python

Python utiliza convenções para indicar acessibilidade:

Convenção Exemplo Descrição
Público atributo Acessível por qualquer código
Protegido _atributo Sinaliza uso interno (convenção)
Privado __atributo Acesso restrito à própria classe
class Conta:
  def __init__(self):
    self.saldo = 0 # Público
    self._senha = "123" # Protegido
    self.__codigo = "X1" # Privado

Implementando Get/Set Tradicionais

Padrão clássico de encapsulamento:

class Pessoa:
  def __init__(self):
    self.__idade = 0 # Atributo privado

  # Método getter
  def get_idade(self):
    return self.__idade

  # Método setter
  def set_idade(self, nova_idade):
    if nova_idade >= 0:
      self.__idade = nova_idade
    else:
      print("Idade não pode ser negativa!")

Uso:

p = Pessoa()
p.set_idade(25)
print(p.get_idade()) # 25
p.set_idade(-5) # Exibe mensagem de erro

Vantagens dos Métodos Get/Set

  • Controle explícito sobre acesso aos dados
  • Validações centralizadas nos métodos set
  • Flexibilidade para alterar implementação interna
  • Compatibilidade com padrões de outras linguagens (Java, C++)

Exemplo com validação:

class ContaBancaria:
  def __init__(self):
    self.__saldo = 0.0

  def get_saldo(self):
    return self.__saldo

  def depositar(self, valor):
    if valor > 0:
      self.__saldo += valor
    else:
      print("Valor inválido!")

  def sacar(self, valor):
    if 0 < valor <= self.__saldo:
      self.__saldo -= valor
      return True
    print("Operação inválida!")
    return False

Exemplo Completo - Produto

class Produto:
  def __init__(self, nome):
    self.__nome = nome
    self.__preco = 0.0
    self.__estoque = 0

  # Getters
  def get_nome(self):
    return self.__nome

  def get_preco(self):
    return self.__preco

  def get_estoque(self):
    return self.__estoque

  # Setters com validação
  def set_preco(self, novo_preco):
    if novo_preco >= 0:
      self.__preco = novo_preco
    else:
      print("Preço não pode ser negativo!")

  def set_estoque(self, quantidade):
    if quantidade >= 0:
      self.__estoque = quantidade
    else:
      print("Estoque não pode ser negativo!")

  def aplicar_desconto(self, percentual):
    if 0 < percentual <= 100:
      self.__preco *= (1 - percentual/100)

Quando Usar Encapsulamento?

Atributos críticos (ex.: saldo bancário, senhas, dados sensíveis)
Validações complexas (ex.: idade ≥ 0, e-mail válido, formato de CPF)
Controle de estado (ex.: um Carro não pode ligar() se já estiver ligado)

Exemplo do Mundo Real:

Um aplicativo de clima:

  • Expõe apenas a temperatura (get_temperatura())
  • Esconde como os dados são obtidos (API externa, cálculos internos)

Benefício: Se a fonte de dados mudar, o código que consome a classe não precisa ser alterado!

Quando Usar Get/Set Tradicionais?

Situações ideais:

  1. Projetos com equipe multidisciplinar (mais explícito)
  2. Integração com outras linguagens
  3. Quando precisa de nomes descritivos para operações complexas
  4. Legados de código que já utilizam esse padrão

Caso de uso típico:

produto = Produto("Notebook")
produto.set_preco(4500)
produto.set_estoque(10)
produto.aplicar_desconto(15)
print(f"{produto.get_nome()}: R${produto.get_preco()}")

Boas Práticas

  1. Use __ para atributos realmente privados
    (Evite acessar _Classe__atributo diretamente)
  2. Documente seus métodos get/set
    (Explique o que fazem e quais validações implementam)
  3. Mantenha a consistência em todo o projeto
    (Padronize os prefixos get/set ou use @property em todo o projeto)
  4. Prefira métodos com nomes significativos quando a operação for complexa
    (Ex.: aplicarDesconto() em vez de setPrecoComDesconto())

Exemplo de documentação:

def get_saldo(self):
  """Retorna o saldo atual da conta
  Returns:
      float: Valor do saldo
  """
  return self.__saldo

Comparação com @property

Get/Set Tradicional @property
Mais explícito Sintaxe mais natural
Verboso Mais conciso
Melhor para operações complexas Ideal para atributos simples
Familiar para devs de outras linguagens Mais pythonico

Escolha baseada em:

  • Padrões do projeto
  • Familiaridade da equipe
  • Complexidade das operações
